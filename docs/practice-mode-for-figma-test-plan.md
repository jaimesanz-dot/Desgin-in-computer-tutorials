## Affected Pages/Routes

- Figma plugin panel — verify the learner sees the current step, hint, progress, validation result, and completion state.
- Figma canvas document state — verify validators inspect selected nodes/subtrees and the `Button / Primary` fallback.

## Key Interactions To Verify

- Launch the plugin and see step 1.
- Click "Check my work" with an empty canvas and receive a clear correction.
- Create a rectangle instead of a frame and receive the frame-specific correction.
- Create the right frame, text, fill, radius, name, and component status across the full lesson.
- Click "Show hint" without losing the current step.
- Reach completion only after the final validator passes.

## Edge Cases

- Wrong node selected.
- Multiple nodes selected.
- No useful node selected, but `Button / Primary` exists.
- Size is close but not exact, returning `near-miss`.
- Text node is missing or nested unexpectedly.
- User edits a previously passed step before completion.
- Validator id is missing or unknown.

## Critical Paths

- End-to-end lesson flow: launch lesson, fail a step, receive correction, fix the document state, pass the step, advance, and reach completion.
- Validator result rendering: `pass`, `fail`, and `near-miss` all render without step-specific UI branching.
- Selection-first validation: selected subtree wins over named fallback, and whole-page scans are not used for normal checks.
